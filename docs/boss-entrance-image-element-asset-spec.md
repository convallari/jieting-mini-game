# 《街亭小游戏》Boss 出场动画——最小图片元素资产规范

> 使用对象：实习动画资产制作师、动画组装人员  
> 本文只定义“需要画哪些静态图片零件”。每张图片只放一个最小元素，不包含时间轴，不把多个可独立运动的物体合在一张图中。

## 1. 什么叫“最小图片元素”

判断方法：如果动画中两个部分需要以不同速度、方向、旋转或透明度运动，就必须拆成两张图片。

例子：

- 旗杆不动、旗面摆动，所以旗杆和旗面必须分开。
- 棋子和棋子落地波纹出现时间不同，所以必须分开。
- 一个“魏”字会被程序复制六次，所以只制作一张“魏”字。
- 三圈波纹扩散速度不同，所以每一圈必须是一张独立图片。
- 角色身体、武器和披风需要不同延迟动作时必须分层；如果现有 Spine 已能控制，则不重新绘制角色身体零件。

严禁：

- 把名字、尘土、旗帜和角色画在同一张 PNG 上。
- 把三个脚印画在一张图片上。
- 把三枚棋子画在一张图片上。
- 把四角墨云画成一张全屏图。
- 在资产中加入血条、提示文字、屏幕震动或棋盘背景。

## 2. 文件格式

### 2.1 必交文件

每个元素交付两份：

1. **可编辑源文件**：优先 `SVG`；包含复杂笔刷或纹理时使用分层 `PSD`。
2. **游戏预览文件**：透明背景 `PNG-32`，颜色空间 `sRGB`，8 bit/通道。

禁止使用 JPG。禁止交只有合成效果、无法修改图层的源文件。

### 2.2 PNG 要求

- RGBA，透明背景。
- 画面外透明像素必须真正为 Alpha 0，不能填白或填黑。
- 边缘保留正常半透明抗锯齿，不能有白边。
- 不预乘黑色背景。
- 导出比例为 1× 逻辑尺寸；不要自行导出 2×、3×。
- 文件方向保持正向，不在 PNG 中预先镜像。

### 2.3 SVG 要求

- `viewBox` 必须与规定逻辑尺寸一致。
- 不嵌入外部链接图片。
- 文字必须转轮廓，不能依赖本机字体。
- 每个文件只有一个最小元素，不把多个备选版本藏在画布外。
- 水墨纹理允许使用蒙版，但蒙版必须内嵌。

## 3. 命名规则

统一格式：

```text
be_{角色}_{动画编号}_{元素序号}_{英文元素名}_{状态}.{扩展名}
```

字段说明：

| 字段 | 可用值 | 示例 |
| --- | --- | --- |
| 固定前缀 | `be` | Boss Entrance |
| 角色 | `com`、`zh`、`feng`、`smy` | 通用、张郃、锋、司马懿 |
| 动画编号 | 两位数字 | `04` |
| 元素序号 | 两位数字，从 `01` 开始 | `02` |
| 英文元素名 | 小写蛇形命名 | `flag_cloth` |
| 状态 | `base`、`alt`、`mask` | 正稿、备选、遮罩 |

正确示例：

```text
be_zh_04_01_flag_pole_base.svg
be_zh_04_01_flag_pole_base.png
be_zh_04_02_flag_cloth_base.svg
be_zh_04_02_flag_cloth_base.png
```

错误示例：

```text
张郃旗子最终版2.png
bossfx.png
flag-all.png
```

同一元素修改时不要增加“最终版”“新版”等字样，使用版本管理记录。文件名保持不变。

## 4. 尺寸、边距与支点记录

- 下表尺寸均为**元素自己的透明画布尺寸**，不是整条动画的 512 × 512 舞台。
- 图形与画布四边至少留 `8 px` 透明边距；带柔光、烟雾的元素至少留 `24 px`。
- 画布不可为了省事全部做成 512 × 512；应按下表裁成合理大小。
- 每张 PNG 旁边必须有同名 `.json`，记录支点和默认用途。

JSON 示例：

```json
{
  "id": "be_zh_04_02_flag_cloth_base",
  "width": 192,
  "height": 144,
  "pivot": { "x": 18, "y": 22 },
  "tintable": false,
  "defaultOpacity": 1,
  "notes": "左上绳结为旋转支点"
}
```

支点含义：

- 旗面：与旗杆连接的位置。
- 角色附属物：与角色身体连接的位置。
- 环、波纹、爆点：几何中心。
- 脚印：脚掌中心。
- 名字和笔锋：画布中心。
- 水纹、虚线：路径起点或图形中心，按表中说明。

## 9. 角色本体的分层原则

张郃、先锋和司马懿已有 Spine 角色资产时，实习生不要重画整套角色。先由负责人确认现有骨骼是否有以下独立插槽：

| 角色 | 必须能独立运动的部分 |
| --- | --- |
| 张郃 | 身体、武器、左右脚、披风或衣摆、左右肩甲 |
| 锋 | 身体、武器、前后脚、披风 |
| 司马懿 | 身体、头部、左右袖、前后衣摆 |

若缺少某个插槽，补图命名使用：

```text
be_{角色}_body_{序号}_{部件名}_base.png
```

例如：`be_smy_body_01_left_sleeve_base.png`。

补图必须与原 Spine 角色在 512 × 512 画布中的比例一致，并在 JSON 中记录与身体连接的支点。没有负责人确认，不得自行替换角色脸、服装或武器造型。

## 10. 目录结构

```text
public/boss-entrance-elements/
  common/
    common_01/
    common_02/
    common_03/
    common_04/
    common_05/
  zhanghe/
    zh_01/ ... zh_09/
  feng/
    feng_01/ ... feng_08/
  simayi/
    smy_01/ ... smy_07/
```

每个目录中只放：

- `.png` 游戏预览图；
- `.svg` 或 `.psd` 可编辑源文件；
- 与 PNG 同名的 `.json` 元数据；
- 一张 `contact-sheet.png` 元素总览图。

PSD 不放进 `public/` 发布目录，正式项目中应存入美术源文件目录；上述结构用于交付包检查。

## 11. contact-sheet 总览图规则

每完成一个小动画目录，制作一张 `contact-sheet.png`：

- 画布 1600 × 900，浅灰底。
- 每个元素显示透明棋盘格背景。
- 元素下方标完整文件名、尺寸、支点坐标。
- 同一元素只出现一次，不展示动画帧。
- 右下角写制作人、日期、审核状态。
- 总览图只用于审核，不进入游戏。

## 12. 实习生逐元素交付流程

1. 先制作当前小动画中的第一个元素。
2. 导出 SVG/PSD、PNG、JSON 三件套。
3. 把 PNG 缩到 64 × 64 检查辨识度和白边。
4. 当前小动画的元素全部完成后制作 contact sheet。
5. 等负责人确认元素拆分和风格。
6. 通过后才进入动画组装，不要提前制作时间轴。

审核意见示例：

```text
元素：be_zh_04_02_flag_cloth_base
结论：修改后复审
1. 旗面必须与旗杆分开，当前 PNG 里还有旗杆残留。
2. 左上绳结支点应为 x=20, y=24。
3. 右边缘增加两处破损，但不要超出 224×160 画布。
4. 删除旗面上的“魏”字，“魏”字使用 03 号独立元素。
```

## 13. 最终验收清单

- 每个可独立运动的部分都已拆开。
- 没有重复绘制程序可以复制、旋转或染色的元素。
- 文件名全部符合 `be_{角色}_{动画编号}_{元素序号}_{名称}_{状态}`。
- PNG、源文件、JSON 一一对应。
- 尺寸与本文表格一致，透明边距符合要求。
- 所有支点在 JSON 中填写，并与 contact sheet 标注一致。
- 名字笔画、旗面、旗徽、旗杆、棋子、波纹均为独立图片。
- 没有棋盘背景、字幕、血条、音效图标或镜头效果混入图片。
- 缩小预览无白边，水墨纹理仍可辨认。
- 负责人确认元素拆分后，才能依据动画制作单制作关键帧。
