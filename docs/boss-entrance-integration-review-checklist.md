# Boss 出场动画接入独立验收清单

依据：`boss-entrance-effects-design.md`、`boss-entrance-image-element-asset-spec.md`、`boss-entrance-animation-asset-production-guide.md`。

> 本次接入验收与正式支持范围为 PC 桌面端（横屏）。移动端 Boss 出场属于未验收的降级路径，不在本次交付承诺内。

## 交付边界

- 29 条时间线均从唯一入口加载；26 条 PNG 序列完整播放。
- `zh_07_boss_step_out`、`feng_06_boss_dash_stop` 作为外部 Spine 引用层；运行时合成对应角色本体。
- 张郃暂使用 `boss1` 入场引用。魏军先锋“锋”没有专属 Spine，使用“锋”字形和水浪序列；禁止用曹操、夏侯惇或其他具名武将替代。
- 司马懿没有专属 Spine 资产，禁止使用曹操或其他武将替代；入场使用墨影图片序列，战斗本体使用“司马懿”字形。

## 状态与时间

- `state.bossEntrance` 是独立状态，不借用普通粒子或暂停菜单生命周期。
- 本局首次遇见播放完整演出（各 Boss 设计时长）；同 Boss 重复生成播放 0.65 秒精简版。
- 0.45 秒后才能跳过；跳过进入 0.35 秒收尾，不是立即销毁。
- 一次演出只允许一个完成入口；连续点击、空格、触摸跳过不会重复完成或死锁。
- 完成后清理帧缓存、一次性事件、输入锁，并恢复演出前暂停状态。

## 战斗冻结与恢复

- 演出期间冻结波次计时、生成计时、敌人位移、Boss 技能冷却、单位攻击、弹道、伤害、持续状态和普通战斗粒子更新。
- UI 绘制、演出时间、必要的 Spine 入场姿势继续更新。
- 开始演出时若正在拖拽，单位安全归还原格/营位并释放 pointer capture。
- 演出期间棋盘、招募、道具、军令和暂停之外的输入被锁定；营寨生命、军粮、汲道等资源 UI 始终可见。
- 若用户原本已暂停，演出结束不能擅自恢复战斗；后台切换与暂停菜单恢复均不丢失演出进度或卡死。

## 演出语义

- 所有路径提示只指向现有左路入口，不出现第二入口。
- 张郃、锋、司马懿分别使用朱红/青蓝/靛黑视觉语法及正确威胁文案。
- `timeline.json.eventFrames` 只触发一次；震屏符合 3px/0.16s 与 1.5px/0.12s 上限。
- 专用音效加载失败时回退 `boss_entrance`，缺音频不能阻塞状态完成。
- 首次显示技能提示；重复演出只显示姓名与技能名。

## 无障碍与布局

- `prefers-reduced-motion: reduce` 禁用快速缩放、残影、震动和强闪，保留淡入、描边、文字与声音提示。
- 桌面横屏与手机竖屏均不遮挡资源 UI、Boss 本体和左路入口。
- `visibilitychange`、窗口缩放、旋转、浏览器暂停恢复后时间不突跳，事件不重复。

## 必需调试入口

- 查询参数可直接触发 3 个 Boss 的首次、重复、可跳过、reduced-motion 场景，无需等待真实波次。
- `window.__jietingDebugState()` 至少暴露：`bossEntrance.active/bossIndex/variant/phase/age/duration/skippable/finishing`、输入锁、演出前暂停状态，以及波次/敌人/伤害快照。
- 建议参数：`?debugBossEntrance=0|1|2&debugBossEntranceVariant=full|repeat&debugState=1`。

## Playwright 最终回归方案

1. 逐 Boss 触发完整演出，采集 0s、0.5s、1.1s、威胁展示、收尾截图及状态快照。
2. 对比演出前/中 `waveTimer`、敌人位置、HP、技能冷却、投射物，确认冻结；结束后确认继续变化。
3. 0.44 秒点击/空格不得跳过；0.46 秒触发后须进入 0.35 秒收尾。
4. 连续快速点击与空格 20 次，确认只完成一次、输入恢复且能继续拖拽。
5. 演出前开始拖拽，再触发演出，确认单位回原位且没有悬空拖拽态。
6. 演出中开关暂停、模拟 `visibilitychange`、调整前后台时钟，确认无突跳/重复事件/死锁。
7. 模拟 `prefers-reduced-motion: reduce`，确认无震屏和高速运动。
8. 1440×900、844×390、390×844 三种视口截图，确认资源 UI、入口和 Boss 可见。
9. 拦截新增音效请求为 404，确认回退音效路径被尝试且演出完成。
10. 最后运行 `npm run build`、`npm run check`、动画强校验和接入静态校验。
