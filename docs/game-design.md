# 街亭小游戏设计文档

## 目标

先做一个尽可能贴近《赵云与阿斗》体验的原创原型：手机竖屏、三国水墨棋盘、塔防路线、营地征兵、拖拽摆放、同级合成、文字将领组合、铲子开垦、馒头经济和强动画反馈。后续主题将收束到“街亭之战”，再做机制和美术创新。

## 录屏观察

- 画面比例为竖屏，录屏分别为 `432x960` 和 `576x1280`，核心 UI 按手机单手操作布局。
- 开局页为水墨山石背景，中间大标题“赵云与阿斗”，五颗星进度，棋盘上的手写字角色有轻微摆动，底部是红褐色大按钮。
- 战斗页顶部显示暂停、馒头数量、关卡/波次文字；中间是大棋盘，底部是“营”横槽和“征兵”按钮。
- 棋盘以硬边方格组织：浅绿/灰绿为土地，白色为可放置格，灰色为不可用格，路线是较暖的土色通道，左右边缘有“斗”和红心血量。
- 单位大多是白色纸片牌，上面用黑色手写字表现“刀 / 枪 / 弓 / 骑 / 张 / 黄”等，右上角用小数字表现等级。
- 拖拽时会出现放大选中态、虚线指引线、目标格高亮、淡白圆形范围提示。
- 营地格被填充时是横向纸片槽；“征兵”按钮带馒头消耗，消耗不足或操作提示会用醒目的黄色描边文字。
- 合成和将领形成时有金色闪光、短暂扩大、纸片抖动/弹跳；高等级或特殊单位有紫色/金色底光。
- 敌人沿路线移动，到“斗”处扣心；受击有小爆点、黑墨/白烟和轻微后仰；死亡掉馒头并飞向左上资源。
- 录屏里出现了图鉴/广告奖励/失败双倍金币弹窗，本项目先删除广告图标和看广告奖励，仅保留普通结算。

## 第一版布局

- 删除原游戏上半屏对手地图，把可用空间全部给玩家地图。
- 顶部状态栏：暂停按钮、馒头、关卡名、波次、斗血量。
- 主区域：`9 x 12` 方格棋盘，路线贯穿地图并在右侧终点攻击“斗”。
- 底部：营地图标、5 个营槽、大号征兵按钮。
- 右下角不放视频广告奖励入口。

## 核心规则

- 敌人按波次从路线起点进入，沿路线移动，抵达终点则扣除斗血量。
- 开垦格可放置士兵；未开垦土地需用铲子拖入后开垦。
- 营地通过“征兵”填充，消耗馒头；已有营槽会被新结果替换。
- 兵种：刀、枪、弓、骑。相同兵种、相同等级拖到一起会合成高一级。
- 将领字：如“赵”“云”。两个正确字按顺序相邻放置后，会合成“赵云”将领，合成前不攻击。
- 地图和营地单位都可拖拽、移动、交换、合成。营地不能攻击，地图上单位可攻击。
- 铲子不能合成，拖到未开垦土地上触发开垦动画。

## 动画规格

- 拖拽开始：单位 `120ms` 放大到 `1.14`，加深阴影，原格保留淡色占位。
- 拖拽移动：目标格每帧跟随，高亮用 `sin` 呼吸，范围圆以 `220ms` 淡入。
- 交换落位：`180ms` 回弹，曲线 `easeOutBack`，落点缩放 `1.08 -> 0.98 -> 1`。
- 合成：两张牌向中心吸附 `160ms`，随后金色环形脉冲 `320ms`，新牌从 `0.65 -> 1.16 -> 1`。
- 征兵：营槽逐个延迟 `60ms` 翻入，纸片从下方 `14px` 滑入并带轻微旋转。
- 开垦：铲子命中后格子 `360ms` 从灰绿变白，撒出 10 个土色粒子。
- 攻击：近战有 `90ms` 前冲和回弹；远程弹道 `220ms` 飞向敌人，带 2-3 帧拖尾。
- 命中：敌人闪白 `80ms`，弹出红色伤害数字，命中点喷 8 个墨点。
- 死亡：敌人缩小淡出 `260ms`，馒头图标从死亡点飞向左上资源 `520ms`。
- 资源数字：增加时 `180ms` 放大到 `1.22` 再回落。
- 斗受击：终点红心抖动，屏幕轻震 `120ms`，震幅 `3px`。

## 当前实现计划

- 使用零运行时依赖的 HTML/CSS/Canvas/JS。
- 开发依赖只用于抽帧和本地预览：`ffmpeg-static`、`vite`。
- 第一版重心在玩法闭环和动画观感，所有美术均为原创 Canvas 绘制，不复制录屏素材。
