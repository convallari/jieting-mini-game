# 街亭小游戏设计文档

## 目标

先做一个尽可能贴近《赵云与阿斗》体验的原创原型：手机竖屏、三国水墨棋盘、塔防路线、营地征兵、拖拽摆放、同级合成、文字将领组合、铲子开垦、馒头经济和强动画反馈。后续主题将收束到“街亭之战”，再做机制和美术创新。

## 录屏观察

- 画面比例为竖屏，录屏分别为 `432x960` 和 `576x1280`，核心 UI 按手机单手操作布局。
- 开局页为水墨山石背景，中间大标题“赵云与阿斗”，五颗星进度，棋盘上的手写字角色有轻微摆动，底部是红褐色大按钮。
- 战斗页顶部显示暂停、馒头数量、关卡/波次文字；中间是大棋盘，底部是“营”横槽和“征兵”按钮。
- 棋盘以硬边方格组织：浅绿/灰绿为土地，白色为可放置格，灰色为不可用格，路线是较暖的土色通道，左右边缘有“斗”和红心血量。
- 单位大多是白色纸片牌，上面用黑色手写字表现“刀 / 枪 / 弓 / 骑 / 张 / 黄”等，右上角用小数字表现等级。
- 拖拽时会出现放大选中态、虚线指引线、目标格高亮、淡白圆形范围提示。
- 营地格被填充时是横向纸片槽；“征兵”按钮带馒头消耗，消耗不足或操作提示会用醒目的黄色描边文字。
- 合成和将领形成时有金色闪光、短暂扩大、纸片抖动/弹跳；高等级或特殊单位有紫色/金色底光。
- 敌人沿路线移动，到“斗”处扣心；受击有小爆点、黑墨/白烟和轻微后仰；死亡掉馒头并飞向左上资源。
- 录屏里出现了图鉴/广告奖励/失败双倍金币弹窗，本项目先删除广告图标和看广告奖励，仅保留普通结算。

## 第一版布局

- 删除原游戏上半屏对手地图，把可用空间全部给玩家地图。
- 顶部状态栏：暂停按钮、馒头、关卡名、波次、斗血量。
- 主区域：`9 x 12` 方格棋盘，路线贯穿地图并在右侧终点攻击“斗”。
- 底部：营地图标、5 个营槽、大号征兵按钮。
- 右下角不放视频广告奖励入口。

## 核心规则

- 敌人按波次从路线起点进入，沿路线移动，抵达终点则扣除斗血量。
- 开垦格可放置士兵；未开垦土地需用铲子拖入后开垦。
- 营地通过“征兵”填充，消耗馒头；已有营槽会被新结果替换。
- 兵种：刀、枪、弓、骑。相同兵种、相同等级拖到一起会合成高一级。
- 攻击范围：刀半径 `1`；枪半径 `2`，可打到正左/右/上/下两格；弓半径 `3`；骑半径 `1.5`，约等于以自身为中心的九宫格范围。普通兵升级不扩大攻击范围。
- 将领字：如“赵”“云”。两个正确字按顺序相邻放置后，会合成“赵云”将领，合成前不攻击。
- 地图和营地单位都可拖拽、移动、交换、合成。营地不能攻击，地图上单位可攻击。
- 铲子不能合成，拖到未开垦土地上触发开垦动画。

## 动画规格

- 拖拽开始：单位 `120ms` 放大到 `1.14`，加深阴影，原格保留淡色占位。
- 拖拽移动：目标格每帧跟随，高亮用 `sin` 呼吸，范围圆以 `220ms` 淡入。
- 拖拽跟随必须优先于装饰动画：拖动中的纸片不播放大幅缩放/旋转，浏览器指针由 canvas 捕获，范围提示只画轻量虚线圆。
- 交换落位：`180ms` 回弹，曲线 `easeOutBack`，落点缩放 `1.08 -> 0.98 -> 1`。
- 合成：两张牌向中心吸附 `160ms`，随后金色环形脉冲 `320ms`，新牌从 `0.65 -> 1.16 -> 1`。
- 征兵：营槽逐个延迟 `60ms` 翻入，纸片从下方 `14px` 滑入并带轻微旋转。
- 开垦：铲子命中后格子 `360ms` 从灰绿变白，撒出 10 个土色粒子。
- 攻击：近战有 `90ms` 前冲和回弹；远程弹道 `220ms` 飞向敌人，带 2-3 帧拖尾。
- 棋盘和营地里的士兵纸片不随待机/攻击动画移动，动画只作用在字形、挂件和攻击覆盖特效上。
- 命中：敌人闪白 `80ms`，弹出红色伤害数字，命中点喷 8 个墨点。
- 死亡：敌人缩小淡出 `260ms`，馒头图标从死亡点飞向左上资源 `520ms`。
- 资源数字：增加时 `180ms` 放大到 `1.22` 再回落。
- 斗受击：终点红心抖动，屏幕轻震 `120ms`，震幅 `3px`。

## 汉字动画资产

汉字单位不再使用系统字体渲染。字体缩放、倾斜和描边无法复刻录屏里的手绘字变形效果，因此本轮把字形拆成两层资产：

原录屏逐帧观察记录见 `docs/original-animation-analysis.md`。该文档优先作为复刻依据；后续实现前应先确认帧表和目标效果，不再直接按推测修改动画代码。

- `src/hanziAssets.js`：每个字的身份、纸片颜色、墨色、倾角、动作关键帧和挂件。
- `src/vectorHanzi.js`：每个字的 SVG-like 矢量笔画。横、竖、撇、捺、弯钩都是独立 Canvas 路径，允许逐笔画错位、拉伸、压扁和加粗。

主逻辑只负责触发 `idle / drag / drop / attack / merge / sleep / enemyWalk / hit / death`，每个字的整体变形参数从资产表采样，实际字形由矢量笔画绘制。

每个汉字资产包含：

- `role`：`blade / spear / bow / cavalry / char / general / enemy / goal / tool`
- `paper / border / ink`：纸片底色、边框、墨色
- `fontScale / baseline / tilt`：沿用为矢量字形缩放、基线和常态倾角参数，不再代表真实字体
- `attachment`：刀芒、枪尖、弓弦、骑兵速度线、将领金环、铲子等挂件
- `motions`：逐帧关键帧，使用 `scaleX / scaleY / skewX / rotate / x / y / glyphScaleX / glyphScaleY / glow / alpha`

当前矢量笔画覆盖：

- `刀 / 枪 / 弓 / 骑`：兵种牌，不使用字体；攻击时字形可随兵器动作产生笔画拉伸。
- `赵 / 云 / 张 / 飞 / 黄 / 忠`：将领字牌，不使用字体；休眠时笔画灰化、轻微下沉。
- `赵云 / 张飞 / 黄忠`：由两个单字矢量资产并排绘制，不使用双字字体。
- `兵 / 贼 / 卒 / 寇`：敌人字，不使用字体；行走和死亡动画直接作用在笔画上。
- `斗`：目标字，不使用字体；地图两侧和开局展示均可由矢量字绘制。

### 2026-07-09 字形复刻精修

上一版矢量字主要依据整体观察手绘，结果偏标准字结构，小尺寸下不贴近录屏。现在改为先从录屏裁切单字，再按裁切图重画。

参考流程：

- 从两段录屏抽出总览帧到 `output/video-frames/`。
- 选取清楚帧 `0f049...` 的 `00:01:20` 和 `5e667...` 的 `00:01:48` 作为第一批重画基准。
- 裁出单字图集 `output/reference/glyph-crops-sheet.png`，用于对照 `刀 / 枪 / 弓 / 骑 / 赵 / 张 / 黄 / 忠 / 兵 / 贼 / 斗`。
- 使用 `tools/make-animation-references.mjs` 从同一位置连续裁切 30fps 动画条，输出到 `output/reference/animation-strips/`。
- 使用 `tools/build-glyph-masks.py` 从裁切图提取墨迹轮廓，生成 `src/glyphMasks.js`。
- 这些参考图只用于观察，不进入成品和 git。

参考解释：

- `营` 中的白纸牌更接近静态基准字形。
- 地图上的同一字可能处在待机、攻击、受击、命中特效覆盖或遮挡中，不能把任意单帧直接当成基础字形。
- 连续帧显示：多数我方字本体比较稳定，明显变化主要来自纸牌轻微弹跳、外部黑色攻击线、白色命中爆点和金色/黄色底光。
- 因此第一阶段复刻采用“稳定省笔字形 + 外部攻击残影 + 命中墨爆”，避免让字本体过度扭曲。
- 对提取质量可靠的字，渲染优先使用 `glyphMasks` 的校准墨形；当前先启用 `刀 / 枪 / 弓 / 骑 / 斗`。复杂字若 mask 噪点太多，暂时保留手写矢量 fallback，避免字形碎裂。
- mask 字形不再整张缩放，而是逐行切片进入变形场：待机时有持续轻微行波，攻击时按字形 rig 叠加横向甩动、弯折、压缩和拖墨残影。这样基础形更接近裁切参考，同时动画不依赖逐笔手调。

本轮重画原则：

- 不追求完整楷体，优先复刻录屏里的小牌省笔墨形。
- `刀`：两到三笔肥黑弯刀形，不再写成规整“刀”。
- `弓`：连续大弯钩，接近录屏里的 S 形弓背。
- `枪`：左侧木旁和右侧仓部都压缩，保留竖杆、斜挑和下部短框。
- `骑`：左马旁密黑，右侧“奇”压缩为横、口框和下钩。
- `斗`：长竖为主体，左两点和底横更接近目标旗字。
- 动画不再平均变形所有笔画；`src/vectorHanzi.js` 为每个字维护 `feature` 关键笔画索引，但攻击时只做轻微呼吸/加粗。明显攻击感交给 `strokeTrail()`、弹道和 `inkSplash()`。

当前字形身份：

- `刀`：录屏中为近白纸底黑字；实现上加短斜刀光，攻击时横向拉伸、纵向压缩、右前斩。
- `枪`：录屏中攻击弹道最清楚，偏长横刺；实现上加木杆与枪尖，后缩蓄力再向目标方向突刺。
- `弓`：录屏中常在后排并发出细黑飞线；实现上加弓弧、弦和箭尾，先拉弓压缩再释放墨线弹道。
- `骑`：录屏中字形最密、黑墨面积大；实现上加速度尘线，攻击时前冲幅度最大。
- `赵 / 云 / 张 / 飞 / 黄 / 忠`：各自有不同倾角、墨色和呼吸节奏；未配对时进入 `sleep` 灰化下沉。
- `赵云 / 黄忠`：金纸底、金边、将领金环，攻击动作使用 `general` 关键帧。
- `张飞`：录屏中有紫色描边/高亮倾向，当前资产使用紫色纸底和紫边。
- `兵 / 贼`：录屏明确出现的敌字池，使用 `enemyWalk` 产生弹跳、倾斜、错相位行走；`卒 / 寇` 作为备用资产暂不默认生成。
- `斗`：目标字，后续可接入受击红波。
- `铲子`：录屏是灰色铁铲图标，不是“铲”字；实现保留图标绘制，资产只负责纸片和动作参数。

当前实现要点：

- `drawUnitCard()` 改为资产驱动，纸片、字形、挂件、等级仍在同一张牌中绘制。
- `drawEnemies()` 改为按敌字资产渲染，敌人死亡延迟 `0.34s` 后移除，期间播放 `death` 字散墨残影。
- 合成时新增 `absorbFx()`，两张参与合成的牌会向中心吸附，然后新牌播放 `merge`。
- 攻击时新增 `strokeTrail()`，按兵种动作生成短墨线/刀枪弓残影。
- 命中和死亡使用 `inkSplash()` 与 `enemyDeathFx()`，保留墨点和字形残影。

## 当前实现计划

- 使用零运行时依赖的 HTML/CSS/Canvas/JS。
- 开发依赖只用于抽帧和本地预览：`ffmpeg-static`、`vite`。
- 第一版重心在玩法闭环和动画观感，所有美术均为原创 Canvas 绘制，不复制录屏素材。
