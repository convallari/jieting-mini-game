## 总体结论

上一版把字拆成多个 mask 部件并逐行变形，方向不适合复刻。原录屏更像以下组合：

1. 静态字形本体相对稳定，尤其是 `刀`、很多 `枪`、`骑` 的清晰帧。
2. 攻击感大量来自外部黑色斜线、弧形拖影、白色命中闪、黄色底部高亮，而不是所有笔画持续大幅扭动。
3. `弓` 是目前观察到最明显的“字本体换姿势”的兵种，更像几张手绘帧离散切换，而不是一张图被连续拉伸。
4. `枪` 在攻击片段里经常被黑色攻击线遮挡，容易误判为字形变形；需要把遮挡帧和清晰字形帧分开。
5. 适合下一步复刻的方式是逐帧/分状态 sprite 资产，而不是运行时复杂 mask 分件。

## 关键片段观察

### `v2-board-qiang-dao-gong`

参考图：

- `output/reference/animation-analysis/v2-board-qiang-dao-gong-contact.jpg`
- `output/reference/animation-analysis/v2-board-qiang-dao-gong-diff.jpg`

逐帧观察：

- `f001-f027`：左侧 `枪` 被强黑色斜线/弧线穿过，像攻击残影或遮挡层；不能直接当作 `枪` 字本体变形。
- `f031-f048`：`枪` 露出稳定竖枪形；`刀` 和 `弓` 在整段里基本保持清晰字形。
- 差分峰值集中在 `f019-f030`，变化区域主要是左侧攻击残影、底部黄色高亮线和右侧少量字形边缘。

复刻含义：

- `枪` 不应做复杂笔画拉伸；应建立“清晰竖枪帧”和“被攻击残影覆盖帧”的状态切换。
- `刀` 以静态字形加短斩击外部线为主。
- `弓` 在这个局部片段变化较小，不能只根据这里定形。

### `v1-board-weapon-cluster`

参考图：

- `output/reference/animation-analysis/v1-board-weapon-cluster-contact.jpg`
- `output/reference/animation-analysis/v1-board-weapon-cluster-diff.jpg`

逐帧观察：

- `f001-f014`：多张牌处于攻击/遮挡阶段，字面上有黑线、白亮和局部遮挡。
- `f018-f048`：整体进入稳定状态，上排 `骑 / 枪 / 刀`、下排 `弓 / 弓 / 枪` 的基础字形基本不再大幅变化。
- 差分图显示变化覆盖多个格子，说明大量运动来自战斗特效和牌面整体状态，不是单字内部独立笔画持续运动。

复刻含义：

- 需要先做“牌面级弹跳 + 外部攻击线 + 命中闪光”的节奏。
- 字本体应该用少量关键帧切换，而不是每帧实时重算复杂形变。

### `v2-top-dao-gong-qi`

参考图：

- `output/reference/animation-analysis/v2-top-dao-gong-qi-contact.jpg`
- `output/reference/animation-analysis/v2-top-dao-gong-qi-diff.jpg`

逐帧观察：

- `刀`：几乎是稳定形，只有牌面轻微位移和特效影响。
- `弓`：本体姿势变化最明显，`f001 / f010 / f022 / f035` 呈现几种不同弓形，像手绘 sprite 帧循环。
- `骑`：字形复杂但稳定性高；明显变化主要和穿过牌面的斜向攻击线绑定。

复刻含义：

- `弓` 应优先做逐帧资产，至少 4 个姿势帧。
- `骑` 应重点复刻外部冲刺线和牌面前冲，不应先拆很多内部笔画。
- `刀` 可先保持静态字形，攻击感交给外部斩击帧。

### 敌人与目标字

参考图：

- `output/reference/animation-analysis/v2-enemy-zei-right-contact.jpg`
- `output/reference/animation-analysis/v1-enemy-bing-dou-contact.jpg`

逐帧观察：

- 敌人片段帧差均值高，主要因为敌人沿路线移动、血条/命中特效变化。

复刻含义：

- 敌人动画应单独分析，不要混入我方兵种字的动作规则。
- `斗` 的重点是受击反馈，不是字本体大幅变形。

## 单字复刻优先级

### 刀

观察：

- 清晰帧中 `刀` 字非常稳定。
- 攻击片段没有看到明显逐笔画独立运动。

建议：

- 先使用静态字形 sprite。
- 攻击时叠加 2-3 帧外部斩击线、牌面轻微前冲、命中闪。

### 枪

观察：

- 清晰静态形有竖枪形和较扁的被遮挡形。
- 早期帧常被黑色斜线覆盖，误看像字形大变。

建议：

- 建立 `idle-clear`、`attack-covered`、`attack-release` 三种帧。
- 不做高成本逐行 mask 形变。

### 弓

观察：

- 是目前最需要逐帧资产的字。
- 多帧中弯弓姿态离散变化明显，不像单张图连续拉伸。

建议：

- 优先切出 4-6 张 `弓` 的逐帧姿势。
- 攻击循环按 `蓄力压弯 -> 最弯 -> 释放 -> 回弹` 组织。

### 骑

观察：

- 基础字形密集，但清晰帧相对稳定。
- 动感更多来自斜向冲刺线、牌面位移和周围特效。

建议：

- 先做静态字形加冲刺外部线。
- 如要拆分，只拆“整体字形/速度线/牌面”三层，不拆到很多内部笔画。

### 斗

观察：

- 目标字本体稳定。
- 需要强化的是受击、血量红心和顶部小装饰反馈。

建议：

- 不进入士兵逐帧复刻优先级。
- 后续单独做受击状态。

## 下一步建议

在继续写实现前，应先让用户确认以下方向：

1. 停止使用运行时部件 mask 变形作为主要复刻方案。
2. 建立兵种逐帧 sprite 资产表，先从 `刀 / 枪 / 弓 / 骑` 四个字开始。
3. 第一批只做 `弓` 的本体逐帧切换，`刀 / 枪 / 骑` 先做静态字形加外部攻击帧。
4. 攻击动画拆成三层：牌面整体运动、字形 sprite 帧、外部攻击/命中特效。
5. 任何实现前先给出每个字的帧表和目标效果图，由用户确认。

## 2026-07-09 第二轮校正记录

这轮先撤销了上一版未经充分对照的“全兵种夸张形变”提交，然后重新对照三张参考图做校正：

- `v2-top-dao-gong-qi-contact.jpg`：`刀` 本体在 1.6 秒内基本稳定；`弓` 本体有明显离散姿势切换；`骑` 的主要变化是右上/上半格横向黑色冲刺线。
- `v2-board-qiang-dao-gong-contact.jpg`：`枪` 前半段看似变形，实际主要是粗黑斜向遮挡层压在字上；约 f031 后回到清晰竖枪。
- `v1-board-weapon-cluster-contact.jpg`：多兵种攻击初段都有黑线/白亮/黄色底线，约半秒后回到稳定字形。

对应实现调整：

- `刀 / 枪 / 骑`：攻击态停止逐行 mask 弯曲和拖影，避免字本体像橡皮一样被拉扯；只保留轻微位移/发光。
- `枪`：攻击覆盖层改为“前段粗黑遮挡，后段白亮释放并露出清晰字”，贴近原片 f001-f031 的变化。
- `骑`：覆盖层改成更横向、集中在上半格的 3 条速度线，而不是大幅拉伸字形。
- `刀`：斩线缩短到字右下区域，避免整张牌被大斜线覆盖。
- `弓`：继续采用离散手绘姿势帧，但帧序改为 `静态 -> 压弯 -> 卷曲 -> 释放 -> 回弹`，减少连续拉伸感。
- `debugAttack=1`：补充一名靠近骑攻击范围的高血量敌人，只用于稳定验证四兵种攻击动画，不影响正式玩法。

仍待进一步精修：

- `弓` 还需要从原片中单独切更高分辨率的同一枚字连续帧，当前只是按 contact sheet 的姿态重绘。
- `枪` 的遮挡层形状还可以继续贴近原片中“像长枪横扫”的黑色块，而不是普通线段。
- `骑` 原片右上速度线与字本体有局部遮挡关系，当前仍是画在字上的简化层。

## 2026-07-09 复刻帧候选策略

用户确认旧方案的问题在于运行时矢量字形和 mask 变形仍然不像原片，尤其笔画节奏和大形变不准确。因此实验台右侧不再显示猜测式矢量动画，改为直接播放专用单字视频裁出的同一帧 spritesheet，作为“复刻帧候选”基线。

当前含义：

- 左侧 `原片帧` 与右侧 `复刻帧候选` 使用同一批专用视频裁帧，并按帧号同步播放。
- 这一步用于确认逐帧资产本身是否可接受，再决定是否把正式游戏里的 `刀 / 枪 / 弓 / 骑` 战斗动画替换为该帧资产或进一步拆笔画重绘。
- 正式玩法在用户确认后接入这批逐帧资产，但只作为透明字形/攻击线图层绘制在游戏自己的卡片上，不再重复绘制录屏里的九宫格、纸面边框和原片等级角标。
